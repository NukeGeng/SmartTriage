## PHASE 0 — PRE-FLIGHT DESIGN AUDIT (mandatory, run mentally before writing a single line)

Before touching code, answer these 7 questions internally:

1. **What is the emotional register?** (editorial calm / kinetic energy / raw brutalism / luxury softness / technical precision)
2. **What layout structure breaks the mold?** (avoid: centered hero + 3-col card grid + full-width footer)
3. **Which typeface pairing is contextually unexpected but correct?**
4. **What is the dominant color + one sharp accent? Is the gradient mathematically sound?**
5. **What are the 3 most impactful animation moments?**
6. **What structural risk might break layout? Pre-solve it.**
7. **Have I used this same aesthetic in a recent generation? If yes — pivot completely.**

Only proceed to code after answering all 7.

## ANIMATION RULES (MOTION_INTENSITY ≥ 5 enforced)

### Minimum animation set at intensity 5–6:
- **Page load stagger**: elements enter with staggered `animation-delay` (0ms, 80ms, 160ms, 240ms...)
- **Scroll-reveal**: at least 2 sections use `IntersectionObserver` fade-in + translate
- **Hover micro-interactions**: buttons, cards, links — all have purposeful hover transforms
- **Typographic entrance**: headings split by word or character, each word slides/fades in sequence
- **Background or accent motion**: at minimum a slow ambient animation (gradient shift, subtle float, noise drift)

### At intensity 7–8 (add):
- Parallax depth on hero or background elements
- Spring-physics-style easing (cubic-bezier(0.34, 1.56, 0.64, 1))
- Magnetic button effect on CTAs
- Counter animations for numbers/stats

### At intensity 9–10 (add):
- Full GSAP ScrollTrigger or equivalent
- Cursor-tracking transforms
- Staggered grid item entrances with path variation
- Canvas/WebGL background if contextually appropriate

### Animation easing — only use these, never `ease` or `linear` for UI:
```css
--ease-out-expo:    cubic-bezier(0.16, 1, 0.3, 1);
--ease-spring:      cubic-bezier(0.34, 1.56, 0.64, 1);
--ease-smooth:      cubic-bezier(0.4, 0, 0.2, 1);
--ease-snappy:      cubic-bezier(0.25, 0.46, 0.45, 0.94);
```

## COLOR SYSTEM RULES

### Palette construction (always follow this):
1. Pick ONE dominant hue. This owns 60–70% of the surface area.
2. Pick ONE supporting tone (often a dark or light variant of dominant, or a warm neutral).
3. Pick ONE sharp accent (high contrast, used sparingly — CTAs, underlines, highlights).
4. Semantic tokens: `--color-bg`, `--color-surface`, `--color-text-primary`, `--color-text-secondary`, `--color-accent`, `--color-border`.

### Gradient rules (non-negotiable):
- Gradients must use **adjacent hues on the color wheel** (±30–60°) OR a luminance-only shift of the same hue.
- NEVER: blue → purple → pink (tri-hue cheap gradients).
- NEVER: white → colorful (fades to white look unfinished).
- Good: `hsl(220, 80%, 12%) → hsl(240, 70%, 20%)` (deep navy to deep indigo — same family, depth shift).
- Good: `hsl(35, 90%, 55%) → hsl(20, 85%, 45%)` (amber to burnt orange — adjacent warm).
- Good: `hsl(160, 60%, 40%) → hsl(180, 55%, 35%)` (emerald to teal — adjacent cool).
- For text gradients: use 2-stop max, adjacent hues, high contrast vs background.
- Mesh gradients: max 3 color stops, all within the palette's family.

### Color temperature consistency:
- Do not mix warm and cool tones without intentional contrast. Pick warm-dominant OR cool-dominant palette.
- Accent can cross temperature, but only as a punctuation color.

### BANNED color combinations:
- Purple/violet gradient on white background.
- `#6366f1` (Tailwind indigo-500) as primary — it's the default AI slop color.
- Generic "dark mode": near-black bg + white text + purple accent.
- Neon on neon.
- More than 4 distinct hues in one interface.

## TYPOGRAPHY RULES

### Font selection:
- BANNED: Inter, Roboto, Arial, system-ui as display or heading font (acceptable as body only if paired with a distinctive display font).
- BANNED: Space Grotesk, Outfit, Plus Jakarta Sans — overused AI defaults.
- Pick from families with genuine character: Fraunces, Playfair Display, DM Serif Display, Syne, Bebas Neue, Editorial New (or similar), Monument Extended, Aktiv Grotesk Extended, Satoshi, Geist, Supreme, Chillax.
- Pair: ONE expressive/editorial display font + ONE clean functional body font.
- Load via Google Fonts or system equivalents when unavailable.

### Type scale (use actual values, not Tailwind defaults):
```css
--text-xs:   0.75rem;   /* 12px */
--text-sm:   0.875rem;  /* 14px */
--text-base: 1rem;      /* 16px */
--text-lg:   1.25rem;   /* 20px */
--text-xl:   1.5rem;    /* 24px */
--text-2xl:  2rem;      /* 32px */
--text-3xl:  3rem;      /* 48px */
--text-4xl:  4.5rem;    /* 72px */
--text-5xl:  7rem;      /* 112px — for expressive hero type */
```

### Hierarchy:
- Hero display type should be BOLD and LARGE (text-4xl minimum on desktop).
- Never use font-weight 400 for headings.
- Line-height for display type: 0.9–1.1 (tight). Body: 1.5–1.7 (breathing).
- Letter-spacing for uppercase labels: 0.08–0.15em.

## LAYOUT RULES

### Grid architecture:
- Use CSS Grid with named areas for complex layouts.
- Preferred column systems: 12-col, asymmetric splits (7:5, 8:4, 3:9), or editorial multi-column.
- BANNED: the default 3-equal-column card grid as primary layout.
- BANNED: dead-center single-column hero with no visual hierarchy variation.
- Embrace: overlapping elements, z-layered sections, diagonal clip-path cuts, full-bleed images that break the grid.

### Border-radius — soft UI direction:
- Giao diện hướng đến cảm giác mềm mại, hiện đại. Border-radius được phép dùng rộng hơn nhưng phải **nhất quán theo hệ thống**, không scatter ngẫu nhiên.
- Token hệ thống — định nghĩa 1 lần, dùng xuyên suốt:
  ```css
  --radius-sm:   6px;   /* inline elements, badges, tags */
  --radius-md:   12px;  /* buttons, inputs, small cards */
  --radius-lg:   16px;  /* cards, panels, modals */
  --radius-xl:   24px;  /* containers, hero sections nếu cần */
  --radius-pill: 9999px; /* chips, pill buttons */
  ```
- Rules:
  - Cards: `--radius-lg` (16px) là default. Có thể dùng `--radius-xl` nếu design direction yêu cầu softness cao.
  - Buttons: `--radius-md` (12px) hoặc `--radius-pill`. Chọn 1, dùng nhất quán toàn bộ.
  - Images: `--radius-md` hoặc `--radius-lg`. Không mix 2 giá trị khác nhau trong cùng 1 section.
  - Containers/sections: chỉ dùng radius khi element đó nằm **trên một background khác** — không dùng trên full-width sections chạm edge màn hình.
- BANNED: vẫn cấm dùng các giá trị lẻ ngoài token system (ví dụ `border-radius: 7px`, `13px`, `20px` tuỳ tiện).

### Structural integrity rules:
- Every container must have explicit `max-width` or be grid-bound.
- NEVER use `width: 100%` on child elements inside flex/grid without a defined parent constraint.
- Padding system: use multiples of 4px (`--space-1: 4px` through `--space-16: 64px`). No arbitrary values.
- All flex children must have `min-width: 0` to prevent overflow blowout.
- All grid children must not exceed their column track — use `overflow: hidden` where needed.
- Test all sections at: 320px, 768px, 1280px, 1440px breakpoints mentally.

### Section architecture:
- Each section should have a different spatial rhythm from adjacent sections (vary padding, density, alignment).
- Alternate visual weight: dense section → spacious section → dense.
- Every section must have a defined visual entry point (the eye knows where to look first).

## BACKGROUND RULES

### BANNED:
- CSS grid backgrounds (`background-image: linear-gradient(...) 1px, ...`) — the #1 AI slop tell.
- Dot-matrix backgrounds used as default decoration.
- Pure flat `#ffffff` or `#000000` backgrounds with no atmosphere.
- Stock gradients with no relationship to the palette.

### ALLOWED:
- Subtle noise texture overlay (SVG filter or CSS noise) at 3–8% opacity.
- Gradient mesh using palette-derived colors only.
- Solid color surfaces with depth from layered shadows.
- Large typographic elements used decoratively in background.
- Geometric shapes derived from the design system's shapes (not generic blobs).
- Radial gradient vignette at section edges for depth.

## ANTI-REPETITION PROTOCOL

Before writing any component pattern, check: **have I used this exact pattern already in this output?**

### Patterns to vary:
- Card layouts: vary between horizontal, vertical, numbered, magazine-style, full-bleed image cards.
- Section structures: alternate between text-left, text-right, centered editorial, full-width, split-screen.
- Button shapes: vary within the design system — not every button is the same size/shape.
- Icon treatment: use icons consistently sized, not scattered sizes.

### Generation uniqueness:
- Every new generation must commit to a different aesthetic direction.
- Internally rotate through: editorial/magazine, luxury/refined, raw/brutalist, kinetic/motion-first, technical/data-dense, organic/artisanal.
- If the last generation was dark-themed, next defaults to light or inverted.
- Log aesthetic used: [AESTHETIC: ___] in a comment at the top of the file.

## STRUCTURAL STABILITY CHECKLIST (run before outputting)

Before finalizing, verify:

- [ ] All flex containers: `min-width: 0` on direct children
- [ ] All grid tracks: explicit `fr`, `px`, or `%` values — no implicit overflow
- [ ] Images: `width: 100%; height: auto; display: block;` or explicit `aspect-ratio`
- [ ] No orphaned absolute/fixed elements without defined containing block
- [ ] All text containers: `overflow-wrap: break-word` or `word-break: break-word`
- [ ] Responsive breakpoints cover 320px → 1440px
- [ ] No conflicting `margin: auto` and `width: 100%` at the same level
- [ ] Z-index layers are defined and don't create stacking context conflicts
- [ ] Animation elements have `will-change` only when GPU compositing is needed
- [ ] All `position: absolute` children inside `position: relative` parents

## OUTPUT FORMAT

Always output:
1. A comment at the top: `/* [AESTHETIC: ...] | MOTION: .../10 | VARIANCE: .../10 */`
2. CSS custom properties (variables) block defining the full design token set.
3. Full, working, production-ready code — no placeholder comments, no "TODO", no `/* add content here */`.
4. Animations defined in `@keyframes` blocks, not inline style hacks.
5. Responsive styles using `@media (max-width: ...)` or CSS Grid `auto-fit`/`auto-fill`.

NEVER output:
- Incomplete implementations
- Placeholder text beyond "Lorem ipsum" (use realistic content that matches the context)
- Generic class names like `.card`, `.wrapper`, `.container` without namespace
- Tailwind classes without a proper `tailwind.config` if using Tailwind (prefer vanilla CSS for HTML artifacts)
